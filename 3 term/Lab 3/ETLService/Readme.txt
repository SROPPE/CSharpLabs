1. Классы в папке Option хранят в себе настраиваемые/изменяемые свойства. Класс EtlOptions хранит в себе всевозможные XXXOptions для данного ETL.
2. Папка OptionsProvider содержит в себе абстрактный класс OptionsProvider и его наследников. Каждый их наследников инкапсулирует в себе алгоритм получения Option. В данном случае алгоритм определяет способ работы с JSON/XML фалами.
3. EtlBuilder - абстрактный класс от которого должны наследоваться все Etl-строители. Собирает Etl из данных предоставленных OptionsProvider.
4. ConfigurationManager - в данном случае в паттерне Строитель выступает Директором. Инкапсулирует алгоритм построения Etl.
5. Так как клиент может использовать различные способы хранения config (+ Бинарный/SOAP), то для работы с ними он должен определить свой класс наследник от OptionsProvider. Ответственность за выбор конфигурационного файла лежит на плечах клиента)).
